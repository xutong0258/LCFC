# LB Front

基于 Vue 3 + TypeScript + Element Plus + Pinia + Vue Router 的现代化前端项目架构。

## 技术栈

- **Vue 3** - 渐进式 JavaScript 框架
- **TypeScript** - JavaScript 的超集，提供静态类型检查
- **Element Plus** - 基于 Vue 3 的组件库
- **Pinia** - Vue 的状态管理库
- **Vue Router** - Vue.js 官方路由管理器
- **Vite** - 下一代前端构建工具
- **ESLint** - 代码质量检查工具

## 项目结构

```
lb-front/
├── public/                 # 静态资源
├── src/
│   ├── components/         # 公共组件
│   ├── views/             # 页面组件
│   ├── router/            # 路由配置
│   ├── stores/            # Pinia 状态管理
│   ├── utils/             # 工具函数
│   ├── types/             # TypeScript 类型定义
│   ├── App.vue            # 根组件
│   └── main.ts            # 入口文件
├── index.html             # HTML 模板
├── vite.config.ts         # Vite 配置
├── tsconfig.json          # TypeScript 配置
└── package.json           # 项目依赖
```

## 开始使用

### 安装依赖

```bash
npm install
```

### 开发环境运行

```bash
npm run dev
```

### 构建生产版本

```bash
npm run build
```

### 预览生产版本

```bash
npm run preview
```

### 代码检查

```bash
npm run lint
```

### 类型检查

```bash
npm run type-check
```

## 功能特性

- ✅ Vue 3 Composition API
- ✅ TypeScript 支持
- ✅ Element Plus 组件库
- ✅ Pinia 状态管理
- ✅ Vue Router 路由管理
- ✅ Vite 快速构建
- ✅ ESLint 代码规范
- ✅ 自动导入组件和API
- ✅ 路径别名配置
- ✅ 环境变量配置
- ✅ Axios 请求封装

## 开发规范

1. 使用 Composition API 编写组件
2. 遵循 TypeScript 类型约束
3. 使用 ESLint 保持代码风格一致
4. 组件命名使用 PascalCase
5. 文件命名使用 kebab-case

## 浏览器支持

现代浏览器和 IE11+
